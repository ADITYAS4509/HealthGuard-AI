A modern AI-powered healthcare decision-support web application interface.

**DESIGN SYSTEM (REQUIRED):**
*   **Theme:** Professional health-tech website look, not a basic dashboard.
*   **Aesthetic:** Clean, modern, premium UI similar to top healthcare SaaS products.
*   **Color Palette:** Green & teal gradient background (health-themed, calming).
*   **Typography:** High contrast text with fully visible fonts. Use **Arial MT** or similar clean sans-serif font.
*   **Components:** Rounded cards, soft shadows, glassmorphism where appropriate.
*   **Layout:** Smooth spacing and alignment for a desktop-first layout.

**Page Structure:**
1.  **Hero Section:** Project title ("AI HealthGuard – Smart Disease & Emergency Predictor"), subtitle explaining real-time AI symptom analysis, and a "Powered by AI" badge. Includes Stats/Feature Cards (50+ Symptoms Supported, AI Risk Engine, Fast Response Time, Live Health Analysis).
2.  **Symptom Selection Panel:** Search bar for symptoms, categorized symptoms list, dedicated separate column/panel listing all symptoms, selected symptoms displayed as removable tags.
3.  **Patient Details Section:** Age input, Severity slider (1–5), Duration (days), City input, and a primary action button ("Analyze Health Condition").
4.  **Analysis Results Dashboard:** Predicted condition (with confidence %), influential symptoms, Risk score (numeric + visual gauge), Risk level badge (Low / Medium / High).
5.  **Insurance Guidance Panel:** Eligibility status, Recommended claim type (OPD / IPD), Step-by-step guidance cards.
6.  **Nearby Hospitals Section:** List of nearby hospitals based on city. Showing Distance, type (Emergency / General), Call / Navigate buttons.
